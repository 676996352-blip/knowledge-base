### 1.2 用Seedance2.0创作语言类shorts

#### 🎯 核心突破

**传统工作流**:
```
脚本 → 分镜图 → 视频 → 剪辑
```

**Seedance 2.0工作流**:
```
角色图 + 场景关键帧 → 直接生成连贯视频
```

#### 🎬 导演思维

| 传统概念 | AI制作对应 | 关键点 |
|---------|-----------|--------|
| **演员定妆照** | 角色参考图 | 不同剧本配不同服装,提升视觉新鲜感 |
| **摄影棚勘景** | 场景关键帧 | 一个场景只需要一个关键帧图 |
| **服化道** | 物品参考图 | 核心道具需单独生成参考图 |

**核心公式**:
> **参考图精准指定 + 分镜秒级控制 + 特写镜头指定**

#### 📝 制作流程

**第一步:获取对标视频并拆解**

1. **下载视频**:
   - 推荐: https://greenvideo.cc/youtube (稳定,视频音频需分开下载)
   - 备选: https://yt5s.biz/zh-cn
   - 付费: 哼哼猫(https://6666.su27.net/)

2. **对标赛道推荐**:
   - ✅ 优先选择: 非AI赛道的真人搞笑形式类脚本
   - ✅ 爆率最高: 有语言的,或自己加语言
   - ✅ 时长: 30s以内
   - ⚠️ 注意: 避免直接照搬复刻,要有创意改编

3. **智能镜头分割**:
   - 导入剪映 → 右键视频 → 智能镜头分割
   - 人工修正两遍:
     - 第一遍: 完全复刻,理解镜头设计
     - 第二遍: 思考微创新调整

**第二步:复刻/改编/设计脚本**

1. **手动拆解脚本骨架**:
   - 在飞书上手写脚本骨架和改编思路
   - 标黄的地方是基于原脚本改动的地方
   - 按照场景+核心故事线描述

2. **使用Google AI Studio生成提示词**:
   - 工具: https://aistudio.google.com/
   - 模型推荐: Gemini 3.1Pro
   - 提示词模版: 使用金鱼的语言类提示词

**金鱼提示词优点**:
1. 会给每个角色生成适配剧本场景的服装
2. 会给到场景、道具全局一致性锚点
3. 对于微创新后的图片还原度、视频动作以及台词对话改编都还不错

**投喂内容**:
1. 对标视频链接/本地视频文件
2. 提示词模版(金鱼的语言类提示词)
3. 人工拆解后的脚本骨架和微创思路
4. 提升图片和视频质量的提示词

**第三步:生成图片(角色图、分镜/关键帧图)**

1. **图片生成工具**:
   - **Nano Banana Pro**: 写实风格断崖领先
   - **Lovart**: Agent,可批量生图
   - 使用平台: https://www.lovart.ai/zh/home

2. **批量生图流程**:
   - 指定基础通用参数
   - 指定图片的参考人物
   - 把Gas给到的【静态起始帧】提示词拿去批量生图

3. **挑选关键帧图**:
   - 原则: 优先选中景为主、能清晰交代环境背景的图
   - 原因: 中景既能展现人物主体,又能包含足够背景信息

**第四步:生成视频提示词、生成视频**

**即梦视频提示词写法**:

**故事类脚本**:
基于AI给的提示词手搓,经验:
1. 语言类尽量结合情景,洗稿还原对标的台词
2. 一些特写镜头手动指定,引导AI聚焦
3. 中近景为主,人物表情渲染力更强,视频画质更清晰
4. 有效增加情绪密度和动作密度
5. 同个场景的多个分镜合并,用秒级写法控制节奏

**稳定核心公式**:
1. **参考图指定**: 清晰使用@符号绑定角色与场景图
2. **分镜秒级控制**: 明确描述每个镜头的时长(如:"0-2秒画面,xxx")
3. **特写镜头指定**: 明确写出"特写镜头",引导AI聚焦

**注意事项**:
- 同一个生成任务中,尽量只包含同一个场景内的剧情
- 每个提交任务,只上传本次生成必须用到的角色图和场景参考图
- 加一句: "根据角色的动作和对话,自由运镜和切镜,中近镜特写。(视频不要出现任何字幕)"

**简单形式类脚本**:
- 直接用官方手册的脚本截图大法
- 参考: 深海圈即梦2.0信息分享汇总帖

#### 💡 关键技巧

**为什么一定要有key frame?**
- 即梦视频参考图片的能力很强
- 可以完美继承关键帧图片角色的服装、角色站位、场景细节、图片的风格
- 用关键帧图可以提前固定住整个视频的风格基调
- 图片抽卡成本远远低于视频抽卡

**key frame怎么选?**
- 核心原则: 优先选中景为主、能清晰交代环境背景的图
- 中景既能展现人物主体的动作或状态,又能包含足够的背景信息
- 它就像电影里的主镜头,奠定了这一段情节的视觉基调

#### 📊 成果展示

**案例: Mira_is_BACK视频**
- 对标视频: 3000万播放
- 复刻视频: 7600万播放
- 制作方法: 跨赛道复刻,改编空间大

### 1.3 用Seedance2.0直出爆款视频

#### 🎬 案例展示

**案例1:周星驰《家有喜事92》经典整蛊脚本复刻**
- 使用Seedance 2.0生成两段15秒的视频
- 后期精调即可

**案例2:Rumi被摸PP超短脚本改造**
- 更简单,生成一段15秒视频
- 后期精调即可

#### 📝 方法分享

**(1) 打开脑洞**
- 不要完全依赖AI给微调方案
- AI给的方案通常会天马行空,不切实际
- AI也给很多人提供了完全一样的方案
- 在对人的情感的把控方面,AI现阶段不可能达到人类的水平
- **要大胆想象,让Seedance 2.0帮我们实现**

**(2) 做好前三秒**
- 整体脉络最重要,决定观众是否会点赞、分享
- 前三秒极其重要,如果节奏拖沓没有看点,观众会划走
- **前三秒关键点**:
  - 信息密度要高
  - 画面要一直有变化
  - 多切换镜头视角和景别
  - 安排更多的动作序列和多变的人物表情

**(3) 后期精调**
- 删减冗余画面
- 调速(一般加快,某些情况慢动作)
- 加音效(更生动)
- 加特效(片段表现力不够时)
- 配符合情景的背景音乐
- 加转场(不同场景间)
- 画面智能调色(剪映自带)

**注意事项**: 导出后用视频播放器播放确认效果

**(4) 多号齐发**
- 同一脚本多账号分发,爆率提升几倍
- 建议做一下对白或部分细节的调整
- 流量这玩意带点玄学,好脚本在不同号表现不同

## 2. 语言类赛道

### 2.1 为什么要做AI语言类shorts

#### 🎯 三大理由

**1. 无语言shorts仍然是基础**
- 生财里无语言类shorts已无门槛
- 不建议通过AI语言类shorts来开YPP(难度较大)

**2. 更大的赛道容量,更强的变现能力**
- 老马教练: "万物皆可解说"
- 张九月: "流量是品味的敌人,品味是单价的朋友"

**RPM对比**:
| 类型 | RPM |
|------|-----|
| 无语言KPOP (1100w播放) | ~0.01-0.03 |
| 语言类KPOP (800w播放) | ~0.09-0.15 |
| **差距** | **5-12倍** |

**语言人群分布**:
- 英语: 美国人、菲律宾等
- 西班牙语: 墨西哥
- 葡萄牙语: 巴西
- 无语言: 印度人最多

**3. 制作难度并不会高**

### 2.2 AI语言类shorts的类型

#### 类型一:轻度对话

**特点**:
- 无语言shorts加上几句简单的旁白、内心戏
- 配音工作量不高

**制作方法**:
1. 视频制作完成后丢给GAS
2. 让GAS结合剧情写适当的台词
3. 使用ElevenLabs、Minimax等配音
4. 丢剪映里剪辑
5. 或使用Veo3、海螺V3.1等模型直接生成音画同步内容

**案例**: 美人鱼脚本
- 开头三句旁白
- 中间一句
- 结尾一句

#### 类型二:中度对话

**特点**:
- 较多的对话,但对话不长
- 语言含量中等

**案例**: 抽象腿视频(500万播放)
- 约30个分镜
- 20个分镜有台词
- 15个分镜对的上口型
- 约110个词

**风险提示**:
⚠️ 名人猎奇视频有很大风险
- DEBUFF叠满: 名人猎奇、密集恐惧、儿童
- 一周五亿流量,颗粒无收

**制作要点**:
- 很片段时长在1s-2.5s范围
- 台词简短,能更好地融入无语言shorts

#### 类型三:重度对话

**特点**:
- 全程对话,语速较快
- 语言含量很高

**案例**: Speed帮助大家但没人帮他
- 约22个分镜
- 220个词
- 一分钟之内说完

**语速要求**:
- 每分钟130-180词
- 比普通人平均语速(60-100词/分钟)快
- 属于正常演讲、快读范畴

**制作要点**:
- 只能在说话同时做简单动作
- 做一些表情变化丰富信息密度

### 2.3 实操步骤

#### 第一步:提取文本,生成音频

**1. 提取文案**:
- 把链接和要求发给GAS
- 或使用哼哼猫等下载软件下载字幕

**2. 更换角色,修改内容**:
- 不做儿童,把爸爸妈妈儿子女儿改成同龄人
- 例如: Sakura→Rumi, Naruto→Abby, Mom→Mira, Dad→Jinu
- 检查修改后的内容是否和情节符合

**3. 导出JSON**:
- 让GAS按要求导出JSON
- 去will大佬的网站生成音频: aihotvideos

**ElevenLabs配音提示词**:
```
# Role: YouTube字幕处理专家 & ElevenLabs架构师 (v17.0 Master)

核心原则:全流程采用双语对照表格形式展示

Phase 1: 角色对话提取与翻译
- 精准角色识别
- 逐字原文提取
- 高质量翻译

Phase 2: 配音脚本标准化
- 格式清洗:添加[Emotion]开头,去除句尾标点
- 互动植入

Phase 3: 最终生产(JSON Production)
```

#### 第二步:生成图片

**方法**:
1. 按照小黄教练的故事板提示词和分镜版来复刻
2. 把提示词复制到飞书表格,导出CSV格式
3. 使用工具: 故事板分镜台词处理.zip
4. 转换成JSON后去aihotvideos生成图片

#### 第三步:生成视频

**提示词模版**: CineDream Architect v12.0 (The Omnibus)

**核心要素**:
- 角色库: Rumi、Mira、Zoey、Jinu、Abby、Mystery
- 绝对禁令: 严禁画外描写、严禁运镜速度词、严禁弱动词
- 运镜手册: 推进、拉远、跟随、环绕、摇移、固定
- 表情与情绪库: 复合情绪词
- 音效与环境库: 自然音效、人造音效、动作音效、环境底噪
- 光影与风格库: Golden hour、Blue hour、Cyberpunk neon等

**核心铁律**:
1. 光栅锁死法则: 所见即所得
2. 6秒动作链: [起始动作] -> [发展动作] -> [结束/反应]
3. 全感官音频协议: 音效与画面精确对应
4. 核心提示词公式: [运镜方式], [视觉主体] + [强动作链], [表情弧光], [环境/光影], [Dialogue], [SFX], [Ambient]

#### 第四步:半自动剪辑

**工具**:
- AutoFlow.zip (谷歌插件)
- 文件夹生成器.zip
- capcut auto-v1.0.0.zip

**配置**:
- 修改config.json里的素材根目录和剪映草稿路径
- 可修改音频倍速等内容

**效果**:
- 最主要的是音频剪了气口,自动对齐
- 视频还是要自行剪辑

### 2.4 小结

**关键要点**:
✅ 相对于无语言shorts,AI语言类制作难度并不大  
✅ 没有那么多分镜,不需要用特别快的节奏  
✅ 只要故事足够吸引人,观众愿意看下去  
✅ 同质化低,不在自己的赛道里卷  
✅ 前三秒依然重要,频道平均选择观看率至少要80%  
✅ 洗稿,不要照抄  
✅ 音效要加上去  
✅ 配音保持频道特色  
✅ 加字幕,导出1080

## 6. 科普解说(拼接类解说)

### 6.1 什么是科普解说

**概念理解**:
- 不是科学科普、地理人文讲解
- 而是"十万个为什么"
- 没见过的东西、现象、事物都可以解说

**拼接类解说**:
- 通过选题采集围绕选题的不同素材
- 通过拼接放在一起的手法
- 属于科普解说的一种

**优缺点**:
| 优点 | 缺点 |
|------|------|
| 基本不会出现0播、低播 | 太稳重,天天万播 |
| 标准化流程后各个视频都是万播 | 流量比爆款视频解说低 |
| 单兵作战比较强劲 | 胜在稳定 |

### 6.2 制作流程

#### 第一步:产出文案

**正练方式**:

**1. 确定选题**:
- 看科普解说类型的对标内容
- 选择10-50个文案丢给克劳德
- 让AI理解并输出这类型的选题是什么

**2. 创造选题**:

**提示词模版**:
```
请帮我创作[数量]个shorts video选题创意,要求如下:

内容结构:
每个选题必须包含:
- 主题标题
- 完整分行文案(150到200英文,200-300字中文)
- 中文关键词5个(用于素材搜索)
- 英文关键词5个(用于素材搜索)
- 选题类型说明(科普/文化/生活等)

文案要求:
- 每句话需要分行书写
- 语言通俗易懂
- 突出新奇性和知识性
- 结尾加入互动引导
- 适合30-50秒短视频呈现
- 文案要先给到完整英文,再给到完整中文

选题标准:
- 话题具有新鲜感和传播性
- 容易获取相关视频素材
- 适合视觉化展示
- 能激发用户评论互动
- 符合shorts video平台调性

关键词标准:
- 关键词要具体且实用
- 符合各平台搜索习惯
- 中英文关键词相对应
- 便于实际素材搜索
- 尽量选择高搜索量词汇

目标受众:[目标年龄段]
```

### 6.3 剪辑优化

#### 第一步:优化转场、气口

**问题**:
- 视频直接素材拼接,加配音
- 素材找的不符合,画面看不清
- 没有转场特效

**解决方案**:
1. 识别字幕
2. 按每句话的停顿添加画面
3. 每句话结束就转场下一个画面
4. 一句话较长,中间多加几个画面
5. 添加转场特效、音效

**效果**: 视频质感上升,流量达到万播标准值

**总结**: 
⚠️ 一定要认真学习手册,一步一步对照着来
⚠️ 不要有自己的想法
⚠️ 如果达不到标准值,回头看看动作是否变形

#### 第二步:优化开头前3s

**YouTube指标**:
- 平均观看百分比
- 选择观看率

**操作步骤**:
1. 把对标视频的开头关键帧截图
2. 谷歌识图功能搜图,找到原视频
3. 下载视频,找到和对标视频同样的片段
4. 看对标视频放了几秒,咱们就跟上
5. 对标用了一个箭头,咱们也用上一个箭头
6. 尽量做到非常相似

**重要提示**:
⚠️ 把开头和对标视频做的一模一样
⚠️ 文案的开头也不改动
⚠️ 这样来提升选择观看率(非常重要)

**成果**:
- 优化后迎来第一个爆款
- 720w观看次数
- 发布一星期才开始爆

**总结**:
⚠️ 不要看到数据不好就焦虑、断更
⚠️ 多从自己身上找原因
⚠️ 坚持更新,等待系统推流

## 7. 爆款视频解说(原视频解说)

### 7.1 什么是爆款视频解说

**定义**: 以爆款视频深度二创去重,或一个博主的多个视频剪辑在一起

**优缺点**:
| 优点 | 缺点 |
|------|------|
| 借鉴的都是爆款视频 | 可能遇到博主diss |
| 二创去重后流量更好 | 版权警告、要赔偿 |
| | 要求删掉视频 |

**规避方法**:
- 要么不以人物为主体
- 要么以国内爆款素材为主体
- 避免很多风险

### 7.2 制作流程

#### 第一步:找到爆款视频

**不需要AI出选题,需要寻找已有爆款视频**

**素材来源**:
- 国内: 抖音、快手
- 海外: 推特、INS、TikTok

**建立素材库**:
- 做爆款视频解说: 抢爆款视频的二创首发机会
- 做拼接类解说: 收集同类型题材的素材

**平台采集**:
- 锚定TikTok、X、INS
- 复选国内抖音、快手
- 国内素材加入文字多,去重去水印麻烦

**了解赛道属性**:
⚠️ 海外人物视频素材做解说面临侵权风险
⚠️ 遇到小气的素材作者可能找麻烦
⚠️ 爆款视频解说优先选择国内爆款视频素材
⚠️ 科普解说拼接类素材首选海外平台

**举例**:
**X平台**:
1. 确定搜索内容如"Animals"
2. 在Top里发现爆款视频
3. 检查是否是同行
4. 如果是内容创作者,关注点赞
5. 挖掘更多作者
6. 第二天打开主页推荐,获得更多内容素材

**不同平台特点**:
- TikTok和INS: 内容和YouTube shorts高度相似,少选择
- 可用作补充素材或监控作者抢二创首发权

**沉淀作者方法**:
- 搜索关键词,关注同类型账号
- 休息一会儿或第二天通过推荐页面刷新
- 最好是新号,容易建立信息茧房
- 不要用网页刷,用手机刷

#### 第二步:准备文案

**① 正练文案**:
- 手动写描述词
- 或把视频抽出图片发给GPT/克劳德
- 让AI描述整体情节
- 套用解说词模版完成文案二次进化

**② 逆练文案**:

#### 第三步:剪辑

**准备素材**:
1. 改写后中文文案
2. 对标视频的原视频素材
3. 改写后配音

**① 导入音频,并识别字幕**
- 导入音频时切气口

**② 导入原视频做处理**

**解说词与视频时长匹配**:
- 解说词比视频长: 调慢原视频速度,或反复复制片段镜像倒放补充时长,或通过类似片段代替
- 解说词比视频短: 删减片段,或加速匹配音频

**③ 处理原视频**
1. 智能场景分割
2. 选择镜像
3. 挨个对每个片段进行居中显示
4. 通过裁剪去除字体符号
5. 画中画效果(背景填充-模糊)
6. 全部处理完后创建复合片段
7. 调整对齐音频时间轴
8. 通过加减速方式

**④ 给复合片段智能调色**
- 智能调色(曲线调整,锐化等)
- 锐化参数
- 色温饱和度色调
- 曲线-亮度-红色通道-蓝色通道-绿色通道
- 原素材音效能提升视频感觉可放出来一些

**⑤ 调整字幕特效**

**关键**: 原素材视频解说的重点

**原因**:
- 原素材要进行视频深度混剪
- 如果有大量使用这个素材的创作者,可能会遇到0播问题
- 没有过重
- 字幕特效是很好的过重方式

**方法**: 在字幕特效上下功夫,根据每句话手动添加emoji表情包

**⑥ 手动添加emoji表情包**

**方法**:
- 通过文案丢给AI提炼每句话emoji关键词
- 对应下载
- emoji素材足够多时就不需要再下载
- 原有的库已经足够使用
